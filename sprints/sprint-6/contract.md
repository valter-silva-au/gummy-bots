# Sprint 6 Contract: Multisensory Feedback

## What Will Be Built
Triple-sensory feedback on every gummy catch: visual particles, ASMR pop audio, and rich haptics.

## Acceptance Criteria
1. Web: Web Audio API generates satisfying pop/bubble sound on catch
2. Web: Different dismiss sound (soft whoosh)
3. Web: Enhanced particle burst with trails and fade
4. Web: "Done" text overlay that fades after catch
5. Mobile: Rich haptic patterns (heavy impact on catch, light on dismiss, notch on miss)
6. Mobile: Audio feedback (pop sound) using Expo AV
7. Mobile: Bot celebration state (sparkle pulse after catch)
8. Both: Smooth state transitions, no audio glitches

## Verification
- Every catch triggers visual + audio feedback
- Sounds feel ASMR-like (soft, satisfying, not harsh)
- Haptic patterns are distinct for catch vs dismiss vs miss
